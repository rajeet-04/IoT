  <action>
    1. Create src/middleware/auth.middleware.js:
       - Import { verifyToken } from '../utils/jwt.js'
       - Export function requireAuth(req, res, next):
         - Extract accessToken from req.cookies.accessToken
         - If no token: return res.status(401).json({ error: 'Authentication required' })
         - Try verifyToken(token), set req.userId = payload.userId, call next()
         - Catch error: return res.status(401).json({ error: 'Invalid or expired token' })
       - Export function optionalAuth(req, res, next): Same but if no/invalid token, set req.userId = null and call next() (for endpoints that work with or without auth)
    2. Create src/routes/auth.routes.js:
       - Import express, zod, User model, hashPassword, verifyPassword, generateAccessToken, generateRefreshToken
       - Define zod schemas:
         - signupSchema: z.object({ body: z.object({ email: z.string().email(), password: z.string().min(8) }) })
         - loginSchema: z.object({ body: z.object({ email: z.string().email(), password: z.string().min(1) }) })
         - forgotPasswordSchema: z.object({ body: z.object({ email: z.string().email() }) })
         - resetPasswordSchema: z.object({ body: z.object({ token: z.string(), newPassword: z.string().min(8) }) })
       - Helper function setAuthCookies(res, accessToken, refreshToken):
         - res.cookie('accessToken', accessToken, { httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: 'lax', maxAge: 15 * 60 * 1000 })
         - res.cookie('refreshToken', refreshToken, { httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: 'lax', maxAge: 7 * 24 * 60 * 60 * 1000 })
       - POST /signup: validate with signupSchema, check email uniqueness, hash password, create User, generate tokens, set cookies, return 201 { user: { id: user._id, email: user.email } }
       - POST /login: validate with loginSchema, find User by email, verify password, generate tokens, set cookies, return 200 { user: { id: user._id, email: user.email } }
       - POST /logout: clear accessToken and refreshToken cookies (res.clearCookie with same options as set), return 200 { message: 'Logged out successfully' }
       - POST /forgot-password: validate with forgotPasswordSchema, find User by email (return 200 even if not found for security), generate random token with crypto.randomBytes(32).toString('hex'), log to console: `PASSWORD RESET TOKEN for ${email}: ${token}`, return 200 { message: 'If an account exists, a reset token has been generated. Check server console.' }
       - POST /reset-password: validate with resetPasswordSchema, find User by email (token includes email or use a simple in-memory token store), verify password meets min length, hash new password, update User.passwordHash, return 200 { message: 'Password reset successfully' }
       - GET /me (protected by requireAuth): find User by req.userId, return 200 { user: { id: user._id, email: user.email, createdAt: user.createdAt } }
       - Export router
    3. Create src/index.js:
       - Import express, dotenv, cors, cookie-parser, connectDB, authRoutes
       - Call dotenv.config()
       - Create express app
       - Middleware order: express.json(), cookie-parser(), cors({ origin: process.env.FRONTEND_URL || 'http://localhost:3001', credentials: true })
       - Routes: app.use('/api/auth', authRoutes)
       - Health check: app.get('/health', (req, res) => res.json({ status: 'ok', uptime: process.uptime(), timestamp: new Date().toISOString() }))
       - Error handler: app.use((err, req, res, next) => { console.error(err); res.status(500).json({ error: 'Internal server error' }) })
       - Connect DB: await connectDB(process.env.MONGODB_URI)
       - Start server: app.listen(process.env.PORT || 3000, () => console.log(`Server running on port ${process.env.PORT || 3000}`))
  </action>
