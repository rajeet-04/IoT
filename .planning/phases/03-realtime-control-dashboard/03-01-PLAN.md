  <acceptance_criteria>
    - src/ws/commandRouter.js exports routeControlCommand(deviceId, action, userId, registry) async function
    - Function generates commandId using crypto.randomUUID()
    - Function verifies device ownership: Device.findOne({ _id: deviceId, userId })
    - If device not found or wrong owner: returns { success: false, error: 'Device not found' }
    - If device not connected: returns { success: false, error: 'Device offline' }
    - If action is "set_relay": validates state parameter (0 or 1), sends {"type":"command","commandId":"<id>","action":"set_relay","state":<0|1>} to ESP32 via registry.get(deviceId).send()
    - If action is unknown: returns { success: false, error: 'Unknown action' }
    - On successful send: returns { success: true, commandId: "<id>" }
    - All operations wrapped in try/catch, errors return { success: false, error: 'Internal error' }
  </acceptance_criteria>
  <action>
    1. Create src/ws/commandRouter.js:
       - Import { randomUUID } from 'node:crypto', Device model
       - Export async function routeControlCommand(deviceId, action, userId, registry, params = {}):
         - Verify ownership: const device = await Device.findOne({ _id: deviceId, userId })
         - If !device: return { success: false, error: 'Device not found' }
         - Verify connection: if (!registry.isConnected(device.deviceId)): return { success: false, error: 'Device offline' }
         - Generate commandId: const commandId = randomUUID()
         - Switch on action:
           - "set_relay":
             - Validate params.state: must be 0 or 1, else return { success: false, error: 'Invalid state value' }
             - const cmd = { type: 'command', commandId, action: 'set_relay', state: params.state }
             - const ws = registry.get(device.deviceId)
             - ws.send(JSON.stringify(cmd))
             - return { success: true, commandId }
           - default: return { success: false, error: `Unknown action: ${action}` }
         - Wrap in try/catch: catch returns { success: false, error: err.message }
  </action>

    - Web client authentication: extracts JWT from cookie or query parameter, verifies with verifyToken
    - If JWT invalid: ws.close(4001, 'Unauthorized')
    - If JWT valid: stores userId on ws object (ws.userId = payload.userId)
    - Message routing distinguishes ESP32 vs web client:
      - ESP32 messages (identified by ws.deviceId): routed to messageRouter (existing behavior)
      - Web client messages (identified by ws.userId): routed to command handler
    - Web client message type "control_command": calls routeControlCommand(msg.deviceId, msg.action, ws.userId, registry, msg.params)
    - Command response sent back to web client: {"type":"command_response","commandId":"<id>","success":true/false,"error":"..."}
    - Web client message type "subscribe_device": adds deviceId to ws.subscribedDevices Set, sends {"type":"subscribed","deviceId":"<id>"}
    - ESP32 state_report and ack messages broadcast to subscribed web clients
    - On web client disconnect: cleans up subscriptions (no DB update needed)
  </acceptance_criteria>
  <action>
    1. Update src/ws/hub.js:
       - Import { verifyToken } from '../utils/jwt.js', { routeControlCommand } from './commandRouter.js'
       - In wss.on('connection', after extracting auth header, add web client detection:
         - Check if auth header is Bearer token (ESP32) or JWT cookie (web client)
         - ESP32 path (existing): validate device token, register with registry
         - Web client path: extract JWT from cookie header (parse "accessToken=<jwt>" from req.headers.cookie), verify with verifyToken(token, JWT_SECRET)
         - If JWT valid: set ws.userId = payload.userId, ws.isWebClient = true, ws.subscribedDevices = new Set()
         - If JWT invalid/missing: ws.close(4001, 'Unauthorized')
       - Update message handler:
         - If ws.isWebClient:
           - Parse message JSON
           - If msg.type === "control_command":
             - const result = await routeControlCommand(msg.deviceId, msg.action, ws.userId, registry, msg.params)
             - ws.send(JSON.stringify({ type: 'command_response', commandId: result.commandId, success: result.success, error: result.error }))
           - If msg.type === "subscribe_device":
             - ws.subscribedDevices.add(msg.deviceId)
             - ws.send(JSON.stringify({ type: 'subscribed', deviceId: msg.deviceId }))
           - If msg.type === "unsubscribe_device":
             - ws.subscribedDevices.delete(msg.deviceId)
         - If ws.deviceId (ESP32):
             - Existing messageRouter behavior
             - Additionally: if msg.type === "state_report" or msg.type === "ack", broadcast to web clients subscribed to this device
       - Add broadcast function: function broadcastToDeviceSubscribers(deviceId, message):
         - Iterate wss.clients, for each client where client.isWebClient && client.subscribedDevices.has(deviceId) && client.readyState === 1: client.send(JSON.stringify(message))
       - On ESP32 state_report: broadcastToDeviceSubscribers(deviceId, { type: 'state_update', deviceId, state: msg.relay, timestamp: new Date().toISOString() })
       - On ESP32 ack: broadcastToDeviceSubscribers(deviceId, { type: 'command_ack', deviceId, commandId: msg.commandId, status: msg.status })
  </action>
